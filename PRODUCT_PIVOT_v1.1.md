# MG-AIOS Product Pivot v1.1

## 核心定位

MG-AIOS 不是另一個 AI。

MG-AIOS 是 ChatGPT 周邊的房仲營運系統。

ChatGPT 是大腦。

MG-AIOS 是記憶、流程、資料庫與整合層。

## 第一原則

如果 ChatGPT 已經做得很好，MG-AIOS 不重做。

MG-AIOS 應該啟動 ChatGPT，而不是取代 ChatGPT。

## ChatGPT 負責

- 思考
- 寫作
- 分析
- 策略
- 腦力激盪
- 行銷文案
- LINE 回覆
- Facebook 貼文
- 591 文案
- 簡報草稿
- 短影音腳本

## MG-AIOS 負責

- 物件資料庫
- 聯絡人資料庫
- 案件旅程
- 工作流程
- 日曆
- 成交流程
- 租賃流程
- 修繕流程
- 文件
- 財務
- 首頁
- 提醒
- 發布狀態
- 版本管理

## 行銷中心原則

行銷中心不呼叫 AI API 產生內容。

每個行銷動作都提供：

「在 ChatGPT 開啟」

流程：

1. MG-AIOS 收集 Property 資料。
2. MG-AIOS 自動產生完整 Prompt。
3. 使用者開啟 ChatGPT。
4. 使用者在 ChatGPT 完成內容。
5. 使用者把結果貼回 MG-AIOS。
6. MG-AIOS 保存最終版本、發布狀態與版本紀錄。

## AI 使用政策

新增任何 AI 功能前，先問：

ChatGPT 是否已經能做好？

如果是，不在 MG-AIOS 內重做。

如果否，才考慮放進 MG-AIOS。

## 成本原則

不要在 ChatGPT Plus 已經能完成的事情上重複花 API 成本。

MG-AIOS 的目標是最大化 ChatGPT Plus，而不是取代它。
